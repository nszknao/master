Detailed Description

Offers methods to use the Conjugate Gradients algorithm for the optimization of models.

As usual with other quasi-Newton methods, the error function is optimized by line searches, but here a special method is used to avoid bad performance that can occurr when using simple successive gradient vectors.

Because of the fact, that usually the target function is not quadratic and line search is not exact, unwished search directions can be calculated. So the algorithm can be restarted during optimization with the search direction set to the current negative gradient.

For a more detailed description of this algorithm please refer to "Neural Networks for Pattern Recognition" by Christopher M. Bishop.

Initializes some internal variables used by the CG algorithm.

The given parameters are used to initialize some variables used by the CG and the line search algorithms and to allocate memory for internal structures.

Parameters:
modelThe currently used model including the parameter vector.
lsType of LineSearch algorithm, that is used by the CG algorithm. Possible values: "Dlinmin" (default value), "Linmin", "Cblnsrch"
resetNo. of iterations that will be performed before a reset of the algorithm takes place. A reset is sometimes necessary because often the line minimization has not finished after "no. of parameters" iterations. If set to "0" (default value), then the algorithm will be reset after iterations, where is the parameter vector.
axInitial left bracket for the line search algorithms, the default bracket is "0".
bxInitial right bracket for the line search algorithms, the default bracket is "1".
lambdaThe initial value for the searched , that will minimize the function used by the line search algorithm. The default value is "0.25".
verboseIf set to "true", a message will displayed when the CG algorithm performs a reset.
Returns:
none

Updates the parameters of the model by using the CG optimization algorithm.

The given input patterns in and their corresponding target values target are used to calculate the error using the given errorfunction of the currently given model . The resulting error function is then minimized by the CG algorithm, the parameters of the model are updated.
Notice: Method init must be called before using the CG algorithm itself.

Parameters:
modelthe currently used model including the parameter vector
errorfunctionfunction used to calculate the error of current model
inInput patterns for the currently used model.
targetTarget values corresponding to the input patterns.
Returns:
the minimum value of the error
